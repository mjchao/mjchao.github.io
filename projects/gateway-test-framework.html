Gateway Test Framework

Summary

Goal:Perform automatic regression tests on a gateway that submits trade orders to an exchange.

Process:

  1. Generate requirements for the gateway.
  2. Design an automated test framework.
  3. Write test cases against requirements.

Delivered:

  • Test framework design document.
  • Automated test framework.
  • Test cases.

Overview

I interned for 10 weeks at Jump Trading during summer 2015. I was part of the Quality Assurance team and received this project (and second one) for my internship. My goal was to test the correctness of a gateway that connected to the BATS exchange.

Process

In this project, I worked with two software engineering interns who wrote the code for the gateway. We discussed the requirements and then validated our findings with our mentors. Then, the software engineering interns implemented the gateway according to the requirements while I implemented my framework to test against the requirements.

During the first week, I familiarized myself with trading terminology and the internal libraries that were available. By the end of the second week, I had designed my framework to use the Command design pattern and interpret test cases written in plaintext. By the end of the sixth week, I had successfully implemented the framework, tested the majority of the requirements and presented it to the team.

The test framework performed various actions such as submitting orders, listening for order confirmations, and killing and restarting the gateway to check for loss of data. The framework was also capable of flooding the gateway with orders and verifying all of them at a later time.
